首页用户名展示

1. 首页用户名展示

方案一: 单独定义接口

前端单独发送一次 ajax 请求. 后端创建接口返回用户信息. 展示到页面上 缺点：需要多发送一次网络请求

<div v-if="username" class="login_btn fl">
    欢迎您：<em>[[ username ]]</em>
    <span>|</span>
    <a href="#">退出</a>
</div>
<div v-else class="login_btn fl">
    <a href="login.html">登录</a>
    <span>|</span>
    <a href="register.html">注册</a>
</div>
方案二: 利用 cookie

后端把数据保存到 cookie 中, 前端访问 cookie 获取信息后加载到页面

优点: 我们可以利用以前的接口, 在返回数据的同时返回cookie.把数据传递到前端. 节省了一次单独的请求.

实现步骤：

在注册或登录后，后端将用户名写入到 cookie 中

前端在加载 index.html 页面时, 从 cookie 中获取用户信息. 进而展示.

前端展示代码如下(这里的代码只是展示, 不用大家写)

<div v-if="username" class="login_btn fl">
     欢迎您：<em>[[ username ]]</em>
    <span>|</span>
    <a href="#">退出</a>
</div>
<div v-else class="login_btn fl">
    <a href="{{ url('users:login') }}">登录</a>
    <span>|</span>
    <a href="{{ url('users:register') }}">注册</a>
</div>

2. 用户信息写入到 cookie

用户登录和注册的时候, 都需要把用户的信息写入到 cookie.

第一部分写入:

在 users.views.py 文件的 RegisterView 类中:

由下列代码: 

# 直接返回响应结果: 
return JsonResponse({'code':0, 
                     'errmsg':'ok'})

改为如下代码: 

# 生成响应对象
response = JsonResponse({'code':0, 
                         'errmsg':'ok'})

# 在响应对象中设置用户名信息.
# 将用户名写入到 cookie，有效期 14 天
response.set_cookie('username', 
                    user.username, 
                    max_age = 3600 * 24 * 14)

# 返回响应结果
return response
第二部分写入:

在 users.views.py 文件的 LoginView 类中:

由下列代码:

# 直接返回响应登录结果:
return JsonResponse({'code':0,
                     'errmsg':'ok'})

改为如下代码: 

# 生成响应对象
response = JsonResponse({'code':0,
                         'errmsg':'ok'})

# 在响应对象中设置用户名信息.
# 将用户名写入到 cookie，有效期 14 天
response.set_cookie('username', 
                    user.username, 
                    max_age = 3600 * 24 * 14)

# 返回响应结果
return response

3. 获取 cookie 信息, 进行页面展示

在 index.js 中:

下面涉及到前端代码, 我们不用添加.

data: {
    // 添加一个变量, 从 cookie 中获取用户信息后
    // 赋值给该变量
    username: '',
},
mounted(){
   // 获取 cookie 中的用户名
   // getCookie 这个方法位于 base.js 文件中.
   this.username = getCookie('username');
}
